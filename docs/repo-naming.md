# Repository Naming Guidelines - StellaXLabs

This document defines the **conventions for naming repositories** under the StellaXLabs organisation.

## General Rules

1. Use **ONLY lowercase letters** while naming repository.
2. Use **ONLY hyphens** (`-`) to separate words in repository name.

## Repository Naming Format

All repositories should follow the format:

`<product-name>-<platform>-<component>`

Where:

- `<product-name>` – the name of the product or project (e.g., `rentio`)
- `<platform>` – the platform or runtime environment the repository targets (e.g., `mobile`, `web`, `iOS`, `Android`)
- `<component>` - the application component, service, or module contained in the repository (e.g., `frontend`, `auth`, `payments`)

This convention ensures **consistency across products** and makes it easy to identify a repository’s **purpose and scope** at a glance.
